## JavaScript / TypeScript Style
- 補助関数、整形関数、変換関数などの名前付き関数は、アロー関数ではなく `function` 宣言を使う
- 例:
  - `function toMonthValue(date: Date) { ... }`
- ループは `for...of` より通常の `for` 文を優先する
- JSX 内の短いイベントハンドラや、配列メソッドに直接渡す短いコールバックはアロー関数でもよい
- 既存ファイルに強い一貫した書き方がある場合は、その書き方を優先する
- 1行で読みにくくなる補助関数は避け、必要なら複数行に分けて書く

## CSS Style
- 新規スタイルは基本的に CSS Modules を使う
- Tailwind のユーティリティはレイアウト補助に留め、見た目の主要ルールは CSS Modules に置く
- 色指定やトーンは既存画面に合わせ、可能な限り `frontend/src/Base.css` の変数を使う
- `style={{ ... }}` の多用は避ける
- 既存の画面に合わせて、角丸・境界線・影・余白のトーンを急に変えない
- 既存デザインシステムやページの雰囲気を崩す独自表現は、明示的に求められた場合だけ行う

## Frontend Editing
- 既存コンポーネントの責務を大きく崩さず、ページ単位の機能追加は新規コンポーネントや新規ページに切り出すことを優先する
- API のレスポンス型は `frontend/src/types/index.ts` に追加し、呼び出しは `frontend/src/api` 配下に分ける
- 役割ごとに UI が分かれる場合は、管理者向けと従業員向けの導線を明確に分ける
